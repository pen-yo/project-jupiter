# Compose 进阶

能用的设计千篇一律，好用的设计万里挑一。

从现在起，你需要控制应用程序在客户端的表现：压榨性能、提高可交互性。

## 懒加载容器

当你有成千上万的数据要在行列容器里展示时，一次性全部渲染会是非常愚蠢的。因为大部分都根本不在视野里，甚至直至进程结束也不会被看见。那么不如**按需**加载（**懒加载**）它们：

```kt
LazyRow {
  item {
    // 前置内容，可选
  }

  items(/* 一维可迭代对象 */) {
    Text("这是第${it}行重复的文本。")
  }

  item {
    // 后置内容，可选
  }
}

// 除此以外还有 LazyColumn、LazyVerticalGrid、LazyHorizontalGrid 等
```
